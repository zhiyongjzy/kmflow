KMFlow

Keyboard & Mouse Flow — 一套键鼠，多台电脑，无缝切换

Rust QUIC Wayland-first 跨平台 MIT

项目定位

用 Rust 构建的跨平台局域网键鼠共享工具（Software KVM），目标是成为 Deskflow/Synergy 的现代替代品。

内存安全 + 高性能
Rust 零开销抽象，无 GC 停顿，无 C/C++ 内存漏洞
QUIC 一体化网络
加密 + 不可靠 datagram（键鼠）+ 可靠 stream（剪贴板）统一在一个协议栈
Wayland-first
原生支持 libei / Input Capture Portal，同时兼容 X11、Windows、macOS
剪贴板内建
文本/图片/文件 URI 剪贴板同步，不需要外部工具拼凑
纯 Rust GUI (iced)
Elm 架构，跨平台一致外观，无 GTK/Qt 依赖
零配置发现
mDNS 自动发现局域网设备，开箱即用

技术栈

层选择理由
语言 Rust 安全、性能、跨平台编译
网络协议 quinn (QUIC) 内建 TLS 1.3 加密；unreliable datagram 传键鼠事件（低延迟）；reliable stream 传剪贴板/文件
序列化 serde + bincode 零拷贝，极低开销
输入捕获 (Linux Wayland) libei + layer-shell freedesktop 标准 + wlroots fallback
输入注入 (Linux Wayland) libei (eis) / wlroots protocols 标准方案
输入 (Linux X11) XInput2 + XTest 成熟稳定
输入 (Windows) Raw Input + SendInput Win32 标准
输入 (macOS) CGEvent tap + post Core Graphics 标准
GUI iced Elm 架构适配 daemon IPC；COSMIC 桌面验证；wgpu 跨平台一致
服务发现 mdns-sd 零配置，自动发现局域网设备
配置 TOML Rust 生态标准

架构设计

kmflow-net
QUIC datagram → 键鼠事件 (不可靠, 低延迟)
QUIC stream → 剪贴板/文件/控制命令 (可靠)
mDNS 自动发现 + 手动 IP
断连检测 (500ms 超时) + 0-RTT 自动重连
休眠/唤醒感知 (监听 logind/IOKit 事件)
kmflow setup-firewall 自动生成 UDP 放行规则
kmflow-input
capture: libei, layer-shell, xinput2, rawinput, cgevent
emulation: libei, wlroots, xtest, sendinput, cgevent
统一 InputEvent enum 抽象
屏幕边缘检测 + 四角死区 (50px, 可配置) 防误触
按键粘滞防护: 焦点切换瞬间，扫描 pressed_keys 集合，向源端发送所有 pending key 的 release 事件
多显示器拓扑: 每条边缘绑定到具体 monitor_id + edge，支持"A 的右屏右边缘 → B"
紧急释放热键 (Ctrl+Alt+Esc) 强制回本机
Wayland 权限: 提供 polkit rule 模板 + systemd 持久授权
kmflow-clipboard
监听本地剪贴板变化
QUIC reliable stream 同步到对端
支持文本 / 图片 / 文件 URI
环路抑制: 每次写入剪贴板附带 origin_peer_id 标记，监听到变化时比对来源，来自远端的写入不回传
kmflow-proto
EventFrame: 序列号 + 时间戳 + 批量事件 (<1200B MTU)
接收端 3-5 帧重排窗口，超时直接投递不等待
ScreenInfo: 分辨率 + scale_factor, DPI 自动缩放 dx/dy
按键传输标准: 传输层统一使用 USB HID scancode，接收端负责映射为本地 keycode（Linux evdev / Win VK / macOS kVK）
默认端口: 4242/udp (QUIC)，可通过 --port 或 TOML port = N 覆盖
握手协议版本协商，不兼容时明确报错提示升级
安全模型
QUIC 内建 TLS 1.3 加密全部流量
TOFU 配对: 首次连接显示公钥指纹，用户确认后存入 known_peers
后续连接自动验证证书，指纹不匹配则拒绝 + 告警
mDNS 仅广播存在，不接受未配对设备的数据连接
延迟预算: 端到端目标 <5ms，拆分为 capture ≤1ms → serialize ≤0.2ms → network ≤2ms (LAN) → deserialize ≤0.2ms → emulate ≤1ms
连接状态机
Disconnected → Discovering → Pairing → Connected → Active
Active → Suspended (休眠) → Reconnecting → Active
Active → Disconnected (超时500ms未恢复 → 释放焦点回本机)

使用流程

P2P 对等模式 — 两台电脑角色对称，不区分 server/client。谁有焦点谁发送，鼠标移回来就自动切换。

典型操作步骤

# 1. 首次安装后，开放防火墙端口 $ kmflow setup-firewall ✓ ufw: allow 4242/udp (QUIC) # 2. 两台电脑都启动 daemon $ kmflow start # 3. 配对（mDNS 自动发现，或手动 IP） $ kmflow pair 192.168.1.42 ? 对方指纹: a3:f2:9c:... 是否信任? [y/N] y ✓ 已保存到 known_peers # 4. 设定屏幕位置关系 $ kmflow layout --right=office-pc # 5. 查看状态 $ kmflow status office-pc 192.168.1.42 connected 2ms 1920x1080@1x # GUI 模式（可选，拖拽配置拓扑） $ kmflow gui # 紧急释放：Ctrl+Alt+Esc 强制焦点回本机
P2P 对等
两边 daemon 角色对称，不需理解 server/client 概念。三台以上也能任意切换。
CLI + GUI 双模式
日常通过 CLI 或 TOML 配置；可选 GUI 拖拽设定拓扑。一个二进制全包含。
鼠标边缘触发
鼠标移到屏幕边缘自动切换到对方电脑，键盘输入跟随焦点，移回来就切回。

  • 项目脚手架：Cargo workspace, CI (GitHub Actions), tracing 日志框架

  • kmflow-daemon: 双机连接，屏幕边缘检测 + 切换（含死区防误触）
  • 焦点切换时 flush 所有 pressed_keys 的 release 事件（防按键粘滞）
  • 紧急释放机制: 组合键强制回到本机（如 Ctrl+Alt+Escape）
  • DPI / 分辨率协商: 连接建立时交换屏幕信息，自动缩放 dx/dy
  • 配对信任: TOFU 模式 + 首次连接指纹确认
  • 基础 CLI 配置 (TOML)
  • 端到端测试: 两台 Linux Wayland 机器间鼠标+键盘共享
  • CI 测试策略: 单元测试 mock libei trait；集成测试用 Linux network namespace 模拟双机 QUIC 通信；真机 Wayland 测试在 self-hosted runner 上跑
Phase 2: 多平台输入后端
  • X11 后端 (capture + emulation)
  • Windows 后端 (capture + emulation)
  • macOS 后端 (capture + emulation)
  • 跨平台按键映射: 传输层统一 USB HID scancode，各平台接收端维护 HID → 本地码映射表 (evdev/VK/kVK)，特别处理 IME/Fn/media 键
Phase 3: 剪贴板 & 服务发现
  • kmflow-clipboard: 剪贴板监听与同步 (文本)，含 origin_peer_id 环路抑制
  • 图片/文件 URI 剪贴板支持
  • mDNS 自动发现局域网设备
  • 防火墙自动配置提示 (ufw/firewalld 规则生成)
Phase 4: GUI & 用户体验
  • kmflow-gui: iced GUI (设备拓扑拖拽配置)
  • 系统托盘 / 状态指示

关键设计决策

为什么选 QUIC 而不是纯 UDP + DTLS？

纯 UDP + DTLSQUIC (quinn)
加密需要单独实现握手内建 TLS 1.3, 0-RTT
键鼠事件✓ UDP 直发✓ QUIC Datagram (同样不可靠低延迟)
剪贴板/文件✗ 需要自己实现可靠传输✓ QUIC Stream (内建可靠传输)
连接恢复需要自己实现✓ 0-RTT 快速重连
实现复杂度高低 (quinn 封装好)

为什么选 iced？

方案评价
GTK4Linux 好但 macOS/Windows 依赖重、观感差
Qt需要 C++ binding，编译复杂
egui即时模式适合开发工具，外观偏"程序员风"，状态管理自由度过高
icedElm 架构天然适配 daemon IPC；COSMIC 桌面验证；wgpu 跨平台一致；强制状态分离

协议设计

// 每个 datagram 包含的帧头 pub struct EventFrame { seq: u32, // 序列号，用于乱序检测 timestamp: u64, // 微秒时间戳 events: Vec<InputEvent>, // 可批量，单帧 < 1200B (MTU) } pub enum InputEvent { MouseMove { dx: f64, dy: f64 }, MouseButton { button: Button, state: ButtonState }, Scroll { dx: f64, dy: f64 }, Key { scancode: u32, state: KeyState }, } // 连接建立时交换的屏幕信息 pub struct ScreenInfo { width: u32, height: u32, scale_factor: f64, // HiDPI 缩放 position: Edge, // 在拓扑中的相对位置 } // scancode 使用 USB HID Usage ID (跨平台统一)，接收端映射为本地 keycode

非目标 (不做)

  • 视频流/远程桌面（不是 VNC/RDP）
  • 游戏手柄共享
  • 公网穿透（只做局域网，公网用户自己套 WireGuard/Tailscale）
